# Substitution Model Serialization Formats

Survey of file formats used to serialize substitution model parameters (rate matrices, equilibrium frequencies, rate scalers) across phylogenetic software. Covers nucleotide and amino acid models. Focused on formats that could inform TreeTime v1's model I/O.

## PAML matrix format (de facto standard for amino acid models)

The PAML triangular format is the closest thing to a cross-tool standard. IQ-TREE, PhyML, RAxML-NG, and MEGA all accept custom amino acid matrices in this format.

Structure: a lower-triangular exchangeability matrix (190 values for 20 amino acids) followed by 20 equilibrium frequencies. Amino acid order: A R N D C Q E G H I L K M F P S T W Y V. Whitespace-delimited, no header.

Example (first 4 rows of the LG matrix, from [ATGC Montpellier](http://www.atgc-montpellier.fr/download/datasets/models/lg_LG.PAML.txt)):

```
0.425093
0.276818 0.751878
0.395144 0.123954 5.076149
2.489084 0.534551 0.528768 0.062556
...
(190 exchangeability values total, lower triangle row by row)

0.079066 0.055941 0.041977 0.053052 0.012937 0.040767 0.071586 0.057337 0.022355 0.062157 0.099081 0.064600 0.022951 0.042302 0.044040 0.061197 0.053287 0.012066 0.034155 0.069147
```

Applicable to nucleotide models: a 4x4 GTR matrix would have 6 exchangeability values (lower triangle) + 4 frequencies = 10 values. No tool currently uses PAML format for nucleotide GTR, but the structure is applicable.

Sources: [PAML manual](https://snoweye.github.io/phyclust/document/pamlDOC.pdf), [GIPhy model collection](https://giphy.pasteur.fr/empirical-models-of-amino-acid-substitution/)

## RAxML-NG model string format

Inline parameter specification with curly braces. Used in `.raxml.bestModel` output and accepted as `--model` input for subsequent runs.

Example (partitioned DNA analysis):

```
GTR{0.200/1.000/2.000/4.000/7.000/1.000}+FO{0.280/0.220/0.230/0.270}+G4m{0.500}, NADH4 = 1-504
HKY{2.500}+FO{0.300/0.200/0.250/0.250}+G4m{1.200}, tRNA = 505-656
GTR{0.150/0.800/1.500/3.200/5.600/1.000}+FO{0.310/0.190/0.240/0.260}+I{0.150}+G4m{0.870}, NADH5 = 657-898
```

Components:

- `GTR{AC/AG/AT/CG/CT/GT}` - 6 substitution rates (slash-separated)
- `+FO{A/C/G/T}` - 4 optimized base frequencies
- `+G4m{alpha}` - gamma shape parameter with 4 rate categories
- `+I{prop}` - proportion of invariant sites
- `+R3{r1/r2/r3}{w1/w2/w3}` - FreeRate model with rates and weights
- Partition assignment after comma: `name = sites`

The `.bestModel` file is the round-trip format: RAxML-NG writes it, RAxML-NG reads it. Human-readable, compact, self-contained per partition. No per-site frequencies.

Source: [RAxML-NG wiki](https://github.com/amkozlov/raxml-ng/wiki)

## 6-digit model code (PhyML, IQ-TREE, HyPhy)

Compact encoding of rate parameter equality constraints for nucleotide models. Each digit represents one of the 6 substitution rate pairs (AC, AG, AT, CG, CT, GT). Equal digits = equal rates.

Examples:

- `000000` = JC69/F81 (all rates equal)
- `010010` = K80/HKY85 (transitions vs transversions)
- `010020` = TN93 (two transition rates, one transversion rate)
- `012345` = GTR/REV (all rates free)

Used as command-line argument (IQ-TREE: `-m 010010`, PhyML: `--model 010010`). Encodes model structure, not parameter values. Combined with separate frequency and rate heterogeneity specification.

Source: [IQ-TREE Substitution Models docs](https://iqtree.github.io/doc/Substitution-Models), [PhyML manual](https://gensoft.pasteur.fr/docs/phyml/3.3.20190909/phyml-manual.pdf)

## MrBayes NEXUS block

Model specification embedded in a `begin mrbayes;` block within NEXUS files. Uses `lset` for model structure and `prset` for priors.

Example (GTR+I+Gamma, partitioned):

```
begin mrbayes;
  charset gene1 = 1-500;
  charset gene2 = 501-1000;
  partition genes = 2: gene1, gene2;
  set partition = genes;

  lset applyto=(1) nst=6 rates=invgamma;
  prset applyto=(1) statefreqpr=dirichlet(1,1,1,1);

  lset applyto=(2) nst=2 rates=gamma;
  prset applyto=(2) statefreqpr=fixed(empirical);
end;
```

Key settings:

- `nst=1` (JC/F81), `nst=2` (K80/HKY), `nst=6` (GTR)
- `rates=equal|gamma|propinv|invgamma`
- `statefreqpr=dirichlet(...)` or `fixed(equal)` or `fixed(empirical)`

Specifies model structure and priors, not parameter values. MrBayes estimates parameters during MCMC. Parameter values appear in the `.p` (parameter) output file.

Source: [MrBayes 3.2 manual](https://gensoft.pasteur.fr/docs/mrbayes/3.2.7/Manual_MrBayes_v3.2.pdf)

## BEAST2 XML

Full model specification as XML elements. Self-describing but verbose. Generated by BEAUti, consumed by BEAST2.

Example (GTR+Gamma):

```xml
<substModel id="gtr" spec="GeneralSubstitutionModel">
  <frequencies id="freqs" spec="Frequencies">
    <parameter id="freqParameter" dimension="4" value="0.25 0.25 0.25 0.25"/>
  </frequencies>
  <parameter id="rateAC" value="1.0" lower="0.0"/>
  <parameter id="rateAG" value="1.0" lower="0.0"/>
  <parameter id="rateAT" value="1.0" lower="0.0"/>
  <parameter id="rateCG" value="1.0" lower="0.0"/>
  <parameter id="rateCT" value="1.0" lower="0.0"/>
  <parameter id="rateGT" value="1.0" lower="0.0"/>
</substModel>
<siteModel id="siteModel" spec="SiteModel">
  <parameter id="gammaShape" value="1.0"/>
  <parameter id="proportionInvariant" value="0.0"/>
  <substModel idref="gtr"/>
</siteModel>
```

Linking across partitions: `<substModel idref="gtr"/>` reuses the same model instance. Independent models per partition use distinct `id` attributes. State file (`.state`) serializes current MCMC parameter values in the same XML structure.

Source: [BEAST2 tutorials](https://taming-the-beast.org/tutorials/Introduction-to-BEAST2/), [BEAST2 SiteModel docs](https://beast2-dev.github.io/hmc/hmc/SiteModel/gammaCategoryCount/)

## HyPhy NEXUS+HYPHY block

HyPhy uses NEXUS files with a private `HYPHY` block for model fits. The `.fit` output files contain full parameter estimates in this format, and can be reopened to restore analyses.

The 6-digit model string is also used for nucleotide models: `010010` for HKY85, `012345` for REV/GTR.

Source: [HyPhy wiki](https://hyphy.org/w/index.php/SelectTemplateModel)

## TreeTime v0 text format

Custom human-readable text. `GTR.__str__()` writes, `GTR.from_file()` reads. Line-by-line string matching.

Example (from `packages/legacy/treetime/treetime/gtr.py:145-170`):

```
Substitution rate (mu): 0.003421

Equilibrium frequencies (pi_i):
  A: 0.2983
  C: 0.1836
  G: 0.1963
  T: 0.3218

Symmetrized rates from j->i (W_ij):
     A  C  G  T
  A  0  1.2  2.5  0.8
  C  1.2  0  0.9  3.1
  G  2.5  0.9  0  0.7
  T  0.8  3.1  0.7  0
```

Stores $\mu$, $\pi$, and $W$ for scalar GTR only. No site-specific variant. Fragile parser (depends on exact label text). Alphabet inferred from frequency labels.

## Comparison

| Format                | Content                        | Round-trip   | Human-readable | Cross-tool                     | Per-site |
| --------------------- | ------------------------------ | ------------ | -------------- | ------------------------------ | -------- |
| PAML triangular       | $W$ + $\pi$                    | Yes (manual) | Partially      | IQ-TREE, PhyML, RAxML-NG, MEGA | No       |
| RAxML-NG model string | $W$ + $\pi$ + $\mu$ + $\alpha$ | Yes          | Yes            | RAxML-NG only                  | No       |
| 6-digit code          | Model structure only           | N/A          | Yes            | PhyML, IQ-TREE, HyPhy          | No       |
| MrBayes NEXUS         | Model structure + priors       | Yes          | Yes            | MrBayes only                   | No       |
| BEAST2 XML            | Full model + priors            | Yes          | Verbose        | BEAST2 only                    | No       |
| HyPhy NEXUS+HYPHY     | Full model fit                 | Yes          | Partially      | HyPhy only                     | No       |
| TreeTime v0 text      | $\mu$ + $\pi$ + $W$            | Yes          | Yes            | TreeTime only                  | No       |
| IQ-TREE `.sitefreq`   | $\pi$ only                     | Yes          | Yes            | IQ-TREE only                   | Yes      |

No format stores all three components ($W$, $\pi$, $\mu$) in a cross-tool standard. PAML triangular comes closest for exchangeability + frequencies but omits the rate scalar. RAxML-NG's model string includes all three in a single line but is tool-specific.

## Relevance for TreeTime v1

TreeTime needs to serialize both scalar GTR (for standard analyses) and site-specific GTR (for the auto-partitioning feature). The existing ecosystem has no format that covers both. Two sibling proposals address this:

- [Scalar GTR serialization](../proposals/model-serialization-scalar.md): JSON primary format with PAML triangular and RAxML-NG model string exports for cross-tool use
- [Site-specific GTR serialization](../proposals/model-serialization-site-specific.md): JSON primary format (same schema, per-site array dimensions) with IQ-TREE `.sitefreq` export

Both use a `format` field (`"treetime-gtr"` vs `"treetime-site-specific-gtr"`) for schema dispatch.
